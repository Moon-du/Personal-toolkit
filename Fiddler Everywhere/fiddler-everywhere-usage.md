# Fiddler Everywhere 使用说明

> 原帖地址：<https://www.52pojie.cn/thread-1948949-1-1.html>

### Windows

- 直接运行，输入Fiddler Everywhere路径（留空使用默认安装路径）。
- 如果软件安装在默认目录，通常可直接使用默认路径
- 如果安装在自定义目录，需要手动指定 `Fiddler Everywhere` 的实际安装路径
<img width="1470" height="767" alt="image" src="https://github.com/user-attachments/assets/4e8a473e-ad9d-42c2-b21e-9d5599a794a0" />


### Linux
- 直接运行，输入Fiddler Everywhere路径（AppImage释放后的squashfs-root）。
<img width="1945" height="1134" alt="image" src="https://github.com/user-attachments/assets/34136d8b-0fdb-4123-903e-08a14dd00a5b" />


### macOS

- 直接运行，输入Fiddler Everywhere路径（留空使用默认安装路径）。MacOS用户请启动一次Fiddler Everywhere再使用补丁，不然MacOS会认为被修改的Fiddler Everywhere是损坏的App。
<img width="1026" height="830" alt="image" src="https://github.com/user-attachments/assets/e55711e9-8b8e-4957-b3c9-26adba4fe6e4" />


## 适用场景

Fiddler Everywhere 常见可用于以下工作：

- 调试前后端接口请求
- 抓取和分析浏览器网络流量
- 排查 HTTPS 请求异常
- 验证 API 响应内容和请求头
- 辅助移动端或桌面端联网问题定位

## 使用建议

- 优先使用官方发布版本和合法授权
- 在企业环境中使用前，确认代理、证书和安全策略要求
- 涉及 HTTPS 解密时，注意本机证书信任设置和合规要求
